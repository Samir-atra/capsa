VAEWrapper#

Note

For more examples of using the VAEWrapper check out capsa/tests/test_vae.py.

class capsa.VAEWrapper(*args, **kwargs)[source]#

Uses Variational autoencoders (VAEs) (Kingma & Welling, 2013) to estimate epistemic uncertainty.

VAEs are typically used to learn a robust, low-dimensional representation of the latent space. They can be used as a method of estimating epistemic uncertainty by using the reconstruction loss MSE(x, x_hat) - in cases of out-of-distribution data, samples that are hard to learn, or underrepresented samples, we expect that the VAE will have high reconstruction loss, since the mapping to the latent space will be less accurate. Conversely, when the model is very familiar with the features being fed in, or the data is in distribution, we expect the latent space mapping to be robust and the reconstruction loss to be low.

We‚Äôre making a restrictive assumption about the prior. Our prior over latent space is a standard unit diagonal gaussian. In other words, the encoder doesn‚Äôt output a full covariance matrix over all dimensions (doesn‚Äôt output a high dim Gaussian).

NOTE: in the VAEWrapper we bottleneck the representation inside the model, reconstruct the input from that low dimensional representation, and use the MSE between the input and its reconstruction as a measure of epistemic uncertainty. However, if the input is already very low dimensional, it‚Äôs unreasonable to talk about bottlenecking this representation even further ‚Äì thus it doesn‚Äôt make sense to use the VAEWrapper with e.g. 1-dim inputs.

Example of usage:
>>> # initialize a keras model
>>> user_model = Unet()
>>> # wrap the model to transform it into a risk-aware variant
>>> model = VAEWrapper(user_model)
>>> # compile and fit as a regular tf.keras model
>>> model.compile(...)
>>> model.fit(...)
